# Contributing to NASClaw

感谢你愿意帮助 NASClaw 变得更好。

这个项目目前最需要的贡献，不一定是大功能，而是 **更真实、更可复现、更易传播的内容**。

## 当前最欢迎的贡献

### 1. NAS 适配经验
请补充不同设备上的部署情况，例如：
- 极空间
- 群晖
- 威联通
- 自建 NAS / Mini PC / 软路由

建议附上：
- 设备型号
- 系统版本
- Docker / Compose 版本
- 是否使用 host network
- 使用的模型
- 成功 / 失败现象

### 2. 文档修正
如果你发现：
- 步骤已经过时
- 命令不适用于某个 NAS
- 某段描述不够准确
- 英文表达不自然

直接提 PR 即可。

### 3. Demo 资产
欢迎补充：
- 首页截图
- GIF 演示
- NAS 控制台截图
- 飞书联动截图
- 架构图

### 4. 踩坑案例
最有价值的内容往往不是“成功了”，而是：
- 为什么失败
- 具体报错是什么
- 最后如何修复

请优先补到 `docs/troubleshooting.md`。

---

## 提交建议

### Issue
适合：
- Bug 报告
- 文档错误
- 机型兼容性问题
- 功能建议

### Pull Request
适合：
- 文档优化
- 配置样例更新
- 安装脚本修正
- Demo / 素材补充

---

## 最小提交规范

请尽量做到：

1. 标题清楚
2. 说明你在哪个 NAS / 系统环境测试过
3. 如果改命令，请说明适用范围
4. 如果修 Bug，请附报错和修复前后差异

---

## 文档风格

NASClaw 的文档偏向：
- 中文优先，尽量直白
- 少抽象，多步骤
- 少“理论正确”，多“用户能跑起来”
- 避免未经验证的夸张结论

如果你写英文文档，请保持：
- 技术表达自然
- 结构和中文版基本对齐
- 不要只做机器直译

---

## 提交前自查

- [ ] 改动是否真的帮助用户更快跑起来
- [ ] 是否避免了未经验证的数据或夸大表述
- [ ] 链接是否可点开
- [ ] 新增文件是否在 README 中有入口
- [ ] 若改动安装命令，是否检查过兼容性

---

## 一个很重要的原则

**这个项目不是为了“写很多文档”，而是为了让用户更少踩坑。**

如果你的改动能减少用户 30 分钟折腾时间，就是高价值贡献。
