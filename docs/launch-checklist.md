# NASClaw 启动前检查清单

> 目标：在正式分享、推广或长期运行前，用 5-10 分钟确认 NASClaw 处于“可上线”状态。

---

## 1. 仓库与文档

- [ ] README 首页能在 10 秒内回答：这是什么、适合谁、为什么值得用
- [ ] README 中的快速开始、飞书配置、踩坑文档链接可打开
- [ ] 中英文 README 至少结构一致，不出现明显断层
- [ ] 关键命令都能直接复制执行，无占位符混淆

## 2. 安装链路

- [ ] `scripts/install.sh` 语法校验通过
- [ ] `docker/docker-compose.yml` 能通过 `docker compose config`
- [ ] `examples/config.example.json` 与文档中的配置说明一致
- [ ] 新用户可在 10-20 分钟内完成第一次启动

## 3. 运行验证

- [ ] Control UI 可访问
- [ ] 至少一个模型可成功响应
- [ ] 数据目录已挂载，重启后配置不丢失
- [ ] 容器设置为异常后自动重启
- [ ] 日志中无持续性报错

## 4. 飞书与工作流联动

- [ ] 飞书应用权限已按文档开通
- [ ] 私聊或群聊消息收发正常
- [ ] 至少验证 1 个写操作，例如文档更新或任务创建
- [ ] 授权失败时能按 troubleshooting 文档定位问题

## 5. NAS 场景稳定性

- [ ] 已确认端口 18789 不冲突
- [ ] 已确认 NAS 磁盘空间充足
- [ ] 已确认是否需要 `host` 网络模式
- [ ] 已确认备份策略，例如备份 `data/` 与 `config/`

## 6. 对外发布前

- [ ] 补齐至少 1 张截图或 GIF
- [ ] 仓库描述、Topics、License 已完善
- [ ] 已准备 1 段 50-100 字项目介绍，便于社区转发
- [ ] 已准备已知限制，避免过度承诺

## 建议的最小上线标准

满足以下 5 条，就可以先上线推广：

1. 文档主链路通顺
2. 安装脚本可跑
3. Docker Compose 可跑
4. 至少一个模型与一个消息渠道可验证
5. 常见故障有文档可查
